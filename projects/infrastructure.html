Project Overview

This is the main career-facing project inside InfraSynth Labs.

Purpose

The infrastructure lab exists to build and document real technical experience. Instead of only describing networking and systems concepts, this project records actual deployment decisions, troubleshooting steps, hardware changes, and future improvements.

The current focus is building a stable network foundation with pfSense before moving into managed switching, virtualization, segmentation, monitoring, and public service isolation.

Milestones

The project roadmap is split into practical phases.

Phase 1 — Public Site Foundation

Build the InfraSynth Labs site, connect GitHub to Cloudflare Pages, and create a public project hub for documentation.

Phase 2 — pfSense Edge Firewall

Deploy pfSense as the primary firewall, establish LAN DHCP/DNS, confirm WAN connectivity, and document the firewall foundation.

Phase 3 — WAN Troubleshooting Documentation

Record the static-IP-through-DHCP behavior, gateway issue, interface checks, and restore process as a real troubleshooting writeup.

Phase 4 — Cisco Switching

Configure the Cisco Catalyst 3750G with hostname, management IP, local login, SSH, and a saved baseline configuration.

Phase 5 — Proxmox Virtualization

Install Proxmox, create the first Linux VM, add internal services, and prepare for future monitoring and Docker workloads.

Phase 6 — Segmentation & Security

Add VLANs, firewall rules, management separation, service isolation, and better visibility across the network.
