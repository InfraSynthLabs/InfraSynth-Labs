Objective

InfraSynth Labs is being built as a living technical portfolio. The purpose of the lab is to design, deploy, troubleshoot, and document real infrastructure instead of only describing theoretical knowledge.

The lab currently focuses on the foundation: replacing ISP gateway routing with pfSense, building a stable internal network, preparing Cisco switching, and planning a future virtualization environment.

Current phase: Build a stable firewall, DNS, DHCP, and documentation foundation before adding VLANs, Proxmox, monitoring, and public services.

Current Status

The lab is in its early infrastructure phase. pfSense is active as the primary firewall, while Cisco switching and Proxmox are staged as upcoming milestones.

Network Plan

The current network is built around pfSense as the central firewall and gateway. Switching, VLANs, and segmentation will be added in controlled phases.

Current Foundation

  • pfSense is deployed as the primary edge firewall.
  • WAN uses DHCP because the static public IP is delivered through ISP DHCP reservation.
  • LAN DHCP is handled by pfSense.
  • DNS foundation is handled through pfSense DNS Resolver.
  • Cloudflare manages public DNS for the InfraSynth Labs domain.

Security Principles

InfraSynth Labs is being designed around controlled exposure, least privilege, segmentation, and careful documentation.

  • Keep management interfaces internal only.
  • Do not expose pfSense, Proxmox, switch management, or server dashboards directly to the internet.
  • Use least-privilege firewall rules when segmentation is introduced.
  • Separate public services from personal devices.
  • Document major changes and troubleshooting steps.
  • Use sanitized examples in public documentation.
  • Avoid publishing real public IPs, MAC addresses, serial numbers, credentials, or sensitive screenshots.

Public documentation rule: never publish live credentials, real WAN IPs, MAC addresses, serial numbers, ISP account details, or screenshots that expose sensitive data.

Next Milestones

The next phase is focused on moving from a basic working network into a documented managed infrastructure environment.

  1. Configure Cisco Catalyst 3750G baseline management.
  2. Assign switch hostname and management IP.
  3. Enable SSH and disable insecure remote access methods.
  4. Install and configure Proxmox.
  5. Create the first Ubuntu Server VM.
  6. Add Docker services.
  7. Design VLAN layout.
  8. Build firewall rules for segmented networks.
  9. Add monitoring and logging.
  10. Document each phase as a portfolio project.
